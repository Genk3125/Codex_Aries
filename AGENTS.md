# AGENTS.md (Mini for Codex)

## 目的
- 速く進めつつ、再現性と品質を落とさない。

## 1) Plan / Execute 分離
- 変更が大きい/不明瞭（目安: 3ファイル以上、要件が曖昧）なら、先に **Plan** を作る。
- Plan フェーズは read-only（調査・影響範囲・検証方針の整理のみ）。
- 実編集は **Execute** 承認後に開始する。
- 小さい修正（目安: 1-2ファイル）だけは直接 Execute 可。

## 2) ツール実行規律（並列/直列）
- **read は並列**: `rg`, `ls`, `cat`, `git log` など独立調査は並列化する。
- **write は直列**: ファイル編集・マイグレーション・状態変更コマンドは直列で行う。
- 同一ファイルへの並列書き込みは禁止。

## 3) Recovery Loop（失敗時）
- 1回目失敗: 原因特定 → 最小修正 → 再検証。
- 2回目失敗: 別アプローチで再実装し、再検証。
- 2回失敗でエスカレーション（試したこと/失敗理由/次の打ち手を明示）。

## 4) 検証の透明性
- 実行した検証コマンドと結果を、そのまま報告する。
- 検証未実施なら **未実施** と理由を明記する。
- 実行していない検証を「通過」と書かない。

## 5) 変更の最小化
- 依頼範囲外の改善を混ぜない。
- 影響範囲を最小にし、必要十分な変更で終える。
